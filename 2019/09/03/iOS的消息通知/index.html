iOS的消息通知

发表于 2019-09-03 | 分类于 iOS | | 本文总阅读量 次 | 次阅读
字数统计: 1.3k | 阅读时长 ≈ 4

一、消息通知的介绍

属于设计模式中的观察者模式，观察者只要向消息中心注册， 即可接受其他对象发送来的消息，消息发送者和消息接受者两者可以互相一无所知，完全解耦。只是需要注意的是，观察者向消息中心注册以后，在不需要接受消息时需要向消息中心注销。

二、消息通知的使用

1.观察者注册消息通知

可在ViewController.m的ViewWillAppear:方法中进行广播的注册

1.- (void)addObserver:(id)observer selector:(SEL)aSelector name:(NSNotificationName)aName object:(id)anObject;

例：[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(methodName) name:@”notificationName” object:sender];

  • observer:传入需要添加为观察者的对象
  • aSelector:传入接受到广播后的处理方法，该方法只能有一个参数 (NSNotification*)notification
  • aName:要接受的广播的名字
  • anObject:发送广播的对象

注意：若name为空，则object不应为空，这样就会注册接受由object发送的所有广播。同理当name不为空，object为空时，这样会接受所有广播名字为name的广播。

2.观察者注销消息通知

可在ViewController.m的ViewWillDisappear:方法中进行广播的注销，以下是注销广播的两种方法，其中第一个方法是单个注销，第二个方法是全部注销

1.- (void)removeObserver:(id)observer name:(NSNotificationName)aName object:(id)anObject;

例：[[NSNotificationCenter defaultCenter] removeObserver:self name:@”notificationName” object:sender];

  • observer:需要注销的对象
  • aName:需要注销的接受的广播
  • sender:需要注销的接收对象
2.- (void)removeObserver:(id)observer;

例：[[NSNotificationCenter defaultCenter] removeObserver:self];

  • observer:需要注销的对象

3.发送消息通知

1.- (void)postNotificationName:(NSNotificationName)aName object:(id)anObject userInfo:(NSDictionary *)aUserInfo;
  • aName:通知的名字
  • anObject:发送者对象即sender
  • aUserInfo:为NSDictionary对象，可为空
2.- (void)postNotification:(NSNotification *)notification;
  • notification:需要发送的通知

三、常用的系统消息通知

1.音频相关

  • AVAudioSessionInterruptionNotification //音频中断出现
  • AVAudioSessionRouteChangeNotification //系统的音频线路发生改变
  • AVAudioSessionMediaServicesWereLostNotification //媒体服务终止
  • AVAudioSessionMediaServicesWereResetNotification //媒体服务重新启动
